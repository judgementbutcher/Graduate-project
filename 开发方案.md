# 基于情感计算的生成式叙事 RPG 系统设计与开发

## 开发方案终稿

## 一、项目定位

本课题将系统定义为一个 **2D RPG 游戏系统**，前端采用 **Godot 4.6** 实现地图、角色、交互和 UI，后端采用 **Python FastAPI** 实现 NPC 决策、情感分析、任务管理和数据服务。Godot 4.6 官方文档已提供完整 4.6 分支文档，Godot 本身面向 2D/3D 游戏开发；FastAPI 则是基于 Python 类型标注的高性能 API 框架，并内置基于 OpenAPI 的交互式文档能力。([Godot Engine documentation](https://docs.godotengine.org/?utm_source=chatgpt.com))

本系统不是单纯做一个 RPG 壳子，也不是简单接一个大模型接口，而是围绕“**情感计算驱动的生成式叙事**”构建一个**系统 + 核心算法**的毕业设计。

------

## 二、课题核心目标

本项目的最终目标分成两部分：

### 1. 系统目标

开发一个可运行的 2D RPG 原型系统，具备以下能力：

- 地图探索
- NPC 对话
- 支线/主线任务推进
- 玩家选择影响 NPC 态度
- NPC 根据情绪、记忆和任务状态动态反馈
- 生成式叙事对话输出

### 2. 算法目标

设计并实现一个可完整讲解的算法：

**情感感知—记忆检索—任务约束融合的 NPC 叙事决策算法**

这个算法是本课题最关键的答辩点。

------

## 三、题目中的“情感计算”是否必须自己训练

## 结论

**不必须自己训练。**

本课题的重点不是“从零训练一个情感模型”，而是设计一套**完整、可解释、可验证**的 NPC 决策机制。
因此，本课题建议采用下面这条主线：

### 推荐主线

- 情感识别模块可以采用：
  - 规则法
  - 预训练模型
  - 轻量分类器
- 课题真正的核心算法是：
  - 情感状态建模
  - 记忆检索
  - 任务约束融合
  - 候选行为评分
  - 叙事输出生成

也就是说：

**“情感计算”是算法中的关键组成部分，但不需要把“自己训练模型”当成整个毕业设计的核心。**

## 四、系统总体设计思路

系统采用**前后端分离**思路。

### 1. 前端：Godot 4.6

负责：

- 2D 地图与场景切换
- 玩家移动
- NPC 交互
- 对话框显示
- 任务面板
- 存档/读档界面
- 向后端发送请求并展示结果

### 2. 后端：FastAPI

负责：

- 接收玩家输入
- 维护 NPC 状态
- 进行情感分析
- 检索历史记忆
- 结合任务约束进行行为决策
- 生成 NPC 回复文本
- 提供存档、任务、关系状态等接口

### 3. 数据层：SQLite + SQLAlchemy

SQLite 官方说明其是**self-contained、serverless、zero-configuration、transactional** 的 SQL 数据库引擎，非常适合单机演示型项目；SQLAlchemy 则提供 Python 常用的数据库工具和 ORM 映射能力，适合把表结构映射成后端对象。([SQLite](https://sqlite.org/about.html?utm_source=chatgpt.com))

## 七、核心算法终稿定义

这里就是你答辩时要重点讲的“完整算法”。

# 算法名称

**情感感知—记忆检索—任务约束融合的 NPC 叙事决策算法**

------

## 八、算法输入与输出

### 输入

在时刻 ( t )，系统输入定义为：

- 玩家输入文本 ( x_t )
- 当前剧情上下文 ( c_t )
- NPC 当前状态 ( s_t )
- NPC 历史记忆集合 ( M )
- 当前任务目标集合 ( G )

其中 ( s_t ) 可包含：

- 情绪状态
- 好感度
- 信任度
- 敌意值
- 警惕值
- 是否掌握某条线索
- 当前剧情阶段

### 输出

算法输出为：

- NPC 最终行为 ( a^* )
- 叙事反馈文本 ( y_t )
- 状态更新结果 ( s_{t+1} )

------

## 九、算法详细流程

## 1. 情感感知

系统先从玩家输入和场景状态中提取情绪特征。

可采用五分类情绪：

- 高兴
- 愤怒
- 悲伤
- 恐惧
- 平静

也可用二维情绪空间：

- valence（正负性）
- arousal（激活度）

可定义情感状态计算式：

[
E_t = \alpha \cdot TextEmotion(x_t) + \beta \cdot EventBias(c_t) + \gamma \cdot RelationBias(s_t)
]

含义：

- `TextEmotion`：从玩家文本中得到情绪倾向
- `EventBias`：剧情事件对情绪的影响
- `RelationBias`：关系状态对情绪反应的修正

------

## 2. 记忆检索

对每条记忆 ( m_i \in M ) 计算相关性：

[
Score_{mem}(m_i) =
\lambda_1 \cdot Sim(q_t,m_i) +
\lambda_2 \cdot Importance(m_i) +
\lambda_3 \cdot Recency(m_i) +
\lambda_4 \cdot EmotionMatch(E_t,m_i)
]

其中：

- `Sim(q_t,m_i)`：当前场景与记忆语义相似度
- `Importance(m_i)`：记忆重要度
- `Recency(m_i)`：时间新近度
- `EmotionMatch(E_t,m_i)`：情感匹配程度

然后取 Top-K 记忆构成当前决策上下文。

------

## 3. 任务约束建模

当前 NPC 的行为不能只看情绪，还必须满足任务逻辑和身份设定。

例如：

- 村长不能轻易泄露终局剧情
- 守卫在未建立信任前不能开门
- 支线任务未触发前不能跳过前置条件
- 某些 NPC 必须优先推进主线

因此要定义约束函数：

[
Constraint(a, G, s_t)
]

用于判定候选行为是否违反：

- 任务阶段
- 世界规则
- 角色身份
- 当前场景限制

------

## 4. 候选行为集合

对 NPC 预设一组行为：

[
A={安慰, 鼓励, 回避, 试探, 拒绝, 提示线索, 发布任务, 降低好感, 发起冲突}
]

这一步非常重要，因为它说明你的系统不是“直接让模型自由发挥”，而是先有**可控的行为空间**。

------

## 5. 决策评分函数

对每个候选行为 ( a ) 进行评分：

[
Score(a) =
w_1 \cdot TaskFit(a,G) +
w_2 \cdot EmotionFit(a,E_t) +
w_3 \cdot MemorySupport(a,M_t) +
w_4 \cdot RelationFit(a,s_t) +
w_5 \cdot NarrativeValue(a) -
w_6 \cdot ConstraintPenalty(a)
]

解释如下：

- `TaskFit`：是否符合当前任务推进逻辑
- `EmotionFit`：是否符合情绪状态
- `MemorySupport`：是否与历史记忆一致
- `RelationFit`：是否符合 NPC 与玩家的关系现状
- `NarrativeValue`：是否能提升剧情表现
- `ConstraintPenalty`：是否违反系统约束

最终输出：

[
a^*=\arg\max_{a\in A} Score(a)
]

------

## 6. 文本生成

行为选出后，再生成文本。

这里有两种实现路线：

### 方案一：模板生成

例如：

- 安慰 + 低信任 + 悲伤
  -> “我不太擅长安慰别人……但你现在看起来确实很难受。”

优点：

- 好控制
- 易实现
- 易答辩

### 方案二：模板 + LLM 润色

先给出结构化结果：

- 行为：提示线索
- 态度：谨慎
- 情绪：中性偏警惕
- 内容：指向北方森林废塔

然后让 LLM 输出自然语言。

优点：

- 文本更自然
- 可展现生成式叙事特征

建议：
**开发时先用模板保证能跑通，再加 LLM 润色作为增强项。**

## 十四、情感计算模块实现建议

你有三种实现路线。

## 路线 A：规则法

做法：

- 建立情感词典
- 设置情绪触发词
- 配合标点、否定词、强度副词
- 输出情绪分值

优点：

- 实现简单
- 可解释性强
- 适合毕业设计初版

缺点：

- 泛化较弱

## 路线 B：轻量文本分类器

做法：

- 收集少量中文情感文本数据
- 用 TF-IDF + 逻辑回归 / 线性 SVM
- 输出情绪分类结果

优点：

- 训练成本低
- CPU 可跑
- 便于展示“我实现了情感识别子模块”

缺点：

- 数据集整理要花时间

## 路线 C：预训练模型直接推理

做法：

- 使用现成情感分析模型
- 不做训练，只做推理

优点：

- 效果通常更稳定
- 工作量低

缺点：

- 容易被老师认为“只是调用模型”

### 最推荐方案

**主线采用 A 或 B，答辩时强调核心算法在决策融合，不在模型训练。**

## 十六、实验与评价设计

毕业设计必须有实验，不然论文会很虚。

建议做 4 类指标。

## 1. 情绪一致性

看 NPC 输出是否符合当前情绪状态。

例如：

- 玩家明显愤怒时，NPC 不应持续高兴闲聊
- 低信任状态下，不应直接交出关键线索

## 2. 叙事合理性

请同学做人工评分，评价：

- 回复是否自然
- 行为是否合理
- 剧情是否连贯

评分区间可设 1–5 分。

## 3. 任务推进正确率

统计：

- 行为选择是否满足任务前后置关系
- 是否出现剧情跳步
- 是否违反设定

## 4. 玩家体验指标

可做问卷：

- 互动真实感
- 沉浸感
- NPC 个性鲜明度
- 选择影响感
